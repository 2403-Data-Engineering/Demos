# Ridgeline Supply — Demo Dataset

## Overview

Fictional office supply retailer used as the demo dataset for Spark Fundamentals training. This dataset is used across all demos from Basic RDD Operations through Data Loading and Saving.

## Files

**stores.csv** — 12 store locations across 4 regions (West, East, Midwest, South), 3 per region. Will be converted to Parquet during class.

**products.csv** — 53 products across 5 categories (Electronics, Office Supplies, Furniture, Breakroom, Software) from 7 fictional suppliers. Will be converted to JSON during class.

**sales_data.csv** — 1,500 transactions spanning all of 2024. Generated by `generate_sales_data.py` with weighted distributions across stores and products and heavier volume on weekdays. Contains approximately 5% error rows.

**generate_sales_data.py** — Python script that produces `sales_data.csv`. Configuration values (row count, error rate, date range, weights) are at the top of the file. Requires `stores.csv` and `products.csv` in the same directory. Standard library only, no extra installs.

## Error Types in Sales Data

The sales data includes intentional errors for use in filtering and cleaning exercises:

- Missing product_id (empty field)
- Missing store_id (empty field)
- Invalid quantity (zero or negative)
- Orphan product_id (references a product not in products.csv)
- Duplicate transaction_id (reuses an earlier ID)
- Malformed rows (extra field, breaks CSV parsing)

## Format Plan

All source files start as CSV. During demos, lookup tables are converted to other formats to introduce format variety naturally:

- stores.csv → Parquet
- products.csv → JSON
- sales_data.csv remains CSV as the primary working dataset

## Notes

Lookup tables are intentionally small (12 stores, 53 products) to support broadcast variable demos in later sessions. The sales data is large enough to demonstrate meaningful filtering and aggregation but small enough to run instantly in a local Docker environment.
